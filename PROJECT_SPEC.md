-   **주요 타겟 오디언스**: 기술 기업 채용 담당자, 프로젝트 매니저

## 3. 사용자 인터랙션 요구사항

-   **애니메이션 수준**: 세련된 페이지 전환 및 요소 등장 효과
-   **반응형 디자인 요구사항**: 모바일, 태블릿, 데스크톱 최적화
-   **접근성 요구사항**: WCAG 2.1 AA 레벨 준수

## 4. 프로젝트 진행 방식

-   **개발 단계**:
    1. 디자인 및 와이어프레임 구성
    2. 프론트엔드 기본 구조 설정 (container-presenter 패턴 적용, container 파일 또는 컴포넌트는 container 네이밍, presenter 파일 또는 컴포넌트는 presenter네이밍)
    3. 주요 컴포넌트 개발(Shadcn ui 사용(latest init))
    4. 백엔드 API 구현
    5. 데이터 연동 및 테스트
    6. 배포 및 최적화
-   **피드백 방식**: 각 단계 완료 후 검토 및 수정
-   **우선순위**: 기능성 > 디자인 > 최적화

## 5. 주요 페이지 상세 명세

### 5.1 홈페이지

-   기능: 자기소개, 주요 프로젝트 하이라이트, 기술 스택 표시
-   레이아웃: 히어로 섹션, 프로젝트 그리드, 기술 아이콘
-   인터랙션: 페이지 이동 간 화면 전환 애니메이션, 스크롤 기반 애니메이션, 부드러운 요소 등장 효과

### 5.2 프로젝트 목록 페이지

-   기능: 모든 프로젝트 표시, 필터링, 정렬
-   레이아웃: 카드 그리드, 필터 사이드바
-   인터랙션: 카드 호버 효과, 필터링 시 애니메이션

### 5.3 프로젝트 상세 페이지

-   기능: 프로젝트 상세 정보, 이미지 갤러리, 사용 기술
-   레이아웃: 설명 섹션, 기술 태그
-   인터랙션: 스크롤 애니메이션

### 5.4 관리자 페이지

-   기능: 로그인, 프로젝트 CRUD, 메시지 관리
-   레이아웃: 대시보드, 입력 폼, 목록 테이블
-   인터랙션: 폼 유효성 검사, 데이터 즉시 반영
